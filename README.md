Customer Churn Analysis System (Synthetic Data)

 Overview:

Customer churn is a major challenge faced by industries like banking, telecom, and subscription-based services. This project focuses on analyzing customer behavior using **synthetic data** to identify patterns and factors that lead to churn.

The system performs **Exploratory Data Analysis (EDA)** to help understand why customers leave and how businesses can reduce churn.

 Problem Statement

Businesses lose revenue when customers stop using their services.
Manual data analysis is:

* Time-consuming
* Inefficient
* Error-prone

Also, real customer data is not easily available due to privacy concerns.
This project solves that by generating **synthetic datasets** and analyzing them.

 Objectives:

The main goal is to analyze customer churn patterns using EDA.

 Key Objectives:

* Generate realistic synthetic customer data
* Clean and preprocess the dataset
* Perform statistical analysis
* Identify churn patterns and trends
* Visualize insights using graphs
* Suggest business improvements

 Dataset Description:

The dataset contains **100,000 synthetic customer records**.

 Features:

* `Customer_ID` – Unique ID
* `Age` – 18 to 70
* `Gender` – Male / Female
* `Tenure` – 0 to 10 years
* `Balance` – 0 to 200,000
* `CreditScore` – 300 to 900
* `EstimatedSalary` – 10,000 to 150,000
* `NumOfProducts` – 1 to 4
* `IsActiveMember` – Yes / No
* `Churn` – 0 (No), 1 (Yes)
